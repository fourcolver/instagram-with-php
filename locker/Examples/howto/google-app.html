Getting Google Client ID

A Google Client ID is required for the following buttons:

  • Google Share
  • Google +1 (works as the Google Share)
  • YouTube Subscribe

If you want to use these buttons, you need to get Google Client ID App for your website. You don't need to get a Client ID if you're not going to use these buttons.

1. Go to the Google Developers Console.

2. Click "Select a project":

3. Click the button with the plus icon:

4. Enter a new project name (for example, your website name) and click "Create":

5. Again click "Select a project":

6. And select your project you have just created:

7. Make sure that you are in the Library section. Find and enable the following APIs:

  • Google+ API (to use the Google Plus and Google Share buttons)
  • YouTube APIs (to use the YouTube Subscribe button)

To enable these APIs, click on a title of the required API in the list and then click the button "Enable".

8. Move to the "Credentials" section:

9. Create new credentials "OAuth client ID"

10. Google may ask you to configure a consent screen before creating OAuth client ID, at this case follow the Google instruction and then return back:

11. Fill up the form:

Field How To Fill
Application Type

Web Application
Authorized Javascript origins

Add the origins (with 'www' and without 'www'):

http://example.com

http://www.example.com

If you use SSL, additionally add URLs with 'https'.
Authorized redirect URIs

Paste your website URL:

http://example.com

12. After clicking on the button Create, you will see your new Client ID:

13. Put your new Google Client ID in settings of your locker.
